A Análise de Interface dos Usuários

A análise de interface dos usuários será baseada em alguns itens que interferem diretamente na usabilidade dos sites:

Informações: O site só precisará conter informações importantes e que sejam necessárias para se atingir o objetivo do cliente. Excesso de informações que não importantes reduz a visibilidade do que que importa, minimizando sua taxa de conversão.

Linguagem: Ao traçar um perfil de clientes usuários, é avaliado a linguagem utilizada no website, que deve ser simples, objetivas e utilizando termos que este perfil esteja familiarizado.

Instruções de uso: Ao navegar no site, o usuário deverá ter disponível instruções para usá-lo, se necessário, não deverá ter que ficar acessando várias páginas para encontrar as informações que precisa.

Padrão: O site deve seguir um padrão em todas as páginas, o usuário não precisa ter que interpretar se as palavras ou ações correspondem a mesma coisa.

Feedback: O site deve retornar informações de forma rápida e coerente sobre a navegação atual e buscas dos usuários.

Saídas de escolhas incorretas: Ao acessar uma página ou selecionar uma ação incorreta, a página deve fornecer ao usuário ferramentas para desfazer a escolha sem precisar de muitos cliques.

Atalhos: O site deve atender à demanda de usuários experientes e também inexperientes. O uso de atalhos pode favorecer os mais experientes, mas também deve ser facilmente interpretado pelas pessoas inexperientes.

Mensagem de Erro: Ao apresentar algum tipo de erro, a página deve retornar o motivo deste erro de forma clara, indicando a solução para a correção do mesmo.

Prevenção de Erro: Ao ser construído, o site deve analisar as circunstâncias que podem levar ao erro, e assim, ao serem detectadas, devem indicar ao usuário as correções necessárias antes de efetivar a ação.

Ajuda: Pode ser que o usuário necessite de ajuda durante a navegação pelo site, portanto, o mesmo deve fornecer mecanismos para auxiliá-lo e tornar a experiência mais agradável possível, retornando exatamente a informação necessária, focada na solução do problema em questão, indicando os passos exatos a serem seguidos de maneira clara e não muito extensa.

Planejamento: O site deve atender aos objetivos do negócio que está sendo apresentando juntamente com as necessidades dos usuários.

Escopo: Será analisado as funcionalidades necessárias para se atingir os objetivos da organização e cliente final.

Fluxo: Será estudado o melhor fluxo a ser seguido para se atingir o objetivo do site, a sequência das páginas interfere diretamente no tempo de permanência do usuário no site.

Estética: As escolhas das cores, imagens e layout é analisada para criar um vínculo entre a identidade visual da empresa com o site, porém de uma forma que a mesma não interfira na experiência do usuário, chamando mais atenção do que a informação em questão.
